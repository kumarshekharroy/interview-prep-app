# Weak Area Log

Use this file as the central source of truth for weak areas. Every repeated mistake must become a specific recovery task.

## Rules

1. Do not write vague entries like "bad at DP."
2. Write the exact failure.
3. Assign a retake task.
4. Add a due date.
5. Close only when evidence exists.

## Severity Levels

| Severity | Meaning | Action |
| --- | --- | --- |
| P0 | blocks interview readiness | recover before priority interview |
| P1 | high-risk common topic | recover within 48 hours |
| P2 | moderate weakness | recover during weekly revision |
| P3 | polish | fix during normal review |

## Closure Criteria

A weak area is closed only when:

* retake score is passing.
* explanation can be given aloud.
* mistake is not repeated in the next mock.
* evidence is linked or summarized.
